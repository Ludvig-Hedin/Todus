# Todus iOS App

Native React Native (Expo Router) mail client for iOS.

## Quick start

```bash
cd /Users/ludvighedin/Programming/personal/mail/apps/ios
pnpm start
```

### Temporary auth bypass (for UI testing)

If login is currently broken and you want to inspect the app quickly, run with:

```bash
EXPO_PUBLIC_AUTH_BYPASS=1 pnpm ios
```

This bypass is in-memory only for the current run and is safe to disable immediately.

To revert to normal auth, run without the flag:

```bash
pnpm ios
```

For iPhone simulator:

```bash
pnpm ios
```

## Configuration

- Default web URL: `https://todus.app`
- Default backend URL: `https://api.todus.app`
- Default app entry URL: `https://todus.app/mail/inbox`
- Default app name: `Todus`
- Override URLs:

```bash
EXPO_PUBLIC_WEB_URL=https://todus.app pnpm ios
EXPO_PUBLIC_BACKEND_URL=https://api.todus.app pnpm ios
EXPO_PUBLIC_APP_NAME=Todus pnpm ios
```

Bundle identifier is set in `app.config.ts`:

- iOS: `com.ludvighedin.todus`

Change this before App Store/TestFlight submission if needed.

## TestFlight flow

Export compliance is preconfigured in `app.config.ts` with `ios.config.usesNonExemptEncryption = false`
so App Store Connect does not ask the encryption questionnaire on each upload (assuming no custom
non-exempt crypto is added).

1. Install EAS CLI once:

```bash
pnpm dlx eas-cli@latest --version
```

2. Login and connect project:

```bash
pnpm dlx eas-cli@latest login
pnpm dlx eas-cli@latest init --idempotent
```

3. Build for internal testing:

```bash
pnpm build:ios:preview
```

4. Build for App Store/TestFlight:

```bash
pnpm build:ios:production
pnpm submit:ios
```

## Notes

- OAuth login opens Google consent screen directly in a WebView (no intermediate web login page).
- Bearer token stored in iOS Keychain via `expo-secure-store`.
- TRPC queries use Bearer auth header for API calls.

## App icon source of truth

- Do not edit `apps/ios/ios/Todus/Images.xcassets/AppIcon.appiconset` manually. The `apps/ios/ios` folder is generated by Expo prebuild and gets overwritten.
- Persistent icon files live in `apps/ios/assets/`:
  - `ios-icon-light.png`
  - `ios-icon-dark.png`
  - `ios-icon-tinted.png`
- iOS icon wiring is configured in `apps/ios/app.config.ts` under `ios.icon`.
- To regenerate native iOS icons after changes:

```bash
cd /Users/ludvighedin/Programming/personal/mail
pnpm --filter @zero/ios exec expo prebuild --platform ios --no-install
```
